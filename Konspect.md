# Общий конспект
DevOps - главная роль - автоматизировать работу разработчиков и вывод продукта к пользователю быстро и без ошибок

CI/CD
CI (Непрерывная интеграция)- запускается после добавления разработчиком кода в репозиторий. Автоматически запускает проект, проводит тесты, ищет уязвимости. Если какие-то неполадки наблюдаются говорит об этом разработчику, если все ок - переходим к CD.
CD (Непрерывная доставка) - код автоматически отправляется в тестовую среду, а потом в копию прода. Чтобы протестировать в реальных условиях без серьезных угроз.

PIPELINE - инструкция как работает СI/CD. Четкий маршрут кода от компа до прода. Не просто набор команд, а визуальное отображение того на каком этапе код и как все поломалось + время выполнения


Iac (Инфраструктура как код)- составляем файл, в котором описываем окружение. Запускаем приложение (например TERRAFORM) и получаем окружение. Таким образом будем запускать его хоть 1000 раз и всегда один результат.

Config Management - запускаем приложение (например ANSIBLE) и описываем в нем в файле все что должно быть в окружении: все библиотеки и зависимости (nginx, curl, git и т.д). Запускаем и получаем этот конфиг на всех серверах запущенных Iac


КОНТЕЙНЕР - стандартизированный пакет с приложением и всем необходимым для его работы

МИКРОСЕРВИСЫ - разбиение большого приложения на маленькие независимые куски. Например каждый микросервис лежит в своем контейнере


ОРКЕСТРАЦИЯ - у нас есть множество контейнеров которыми надо управлять. Чтобы не делать этого вручную мы ставим оркестратор ( НАПРИМЕР KYBERNETES), прописываем в файлах настройки что мы хотим делать.

МОНИТОРИНГ - постоянное наблюдение за работой сервисов. Видим проблемы заранее

AGILE - Методология при которой задачи делятся на части

Scrum - Дополнение к Agile когда все задачи делятся на спринты(1-4 недели) и в конце каждого проводится анализ.
 
# GIT
git init – создание .git в папке

git status – статус изменений в папке

git add «.» - индексация изменения

git commit -m «» - коммит с комментарием

git remote add origin https://github.com/USERNAME/REPOSITORY.git  - добавление ссылки на репозиторий

git push -u origin main – запушить изменения на github

git switch (-c) «» - переключиться на ветку. -с – для создания ветки

git push origin --delete «» - Удалить ветку на github

git pull origin main - поддятнуть изменения с github в локальный репозиторий.
